# PROJECT CONTEXT

## Название
Telegram Content Pipeline

## Цель
Автоматически собирать новости из Telegram-каналов, превращать их в полезные финансовые посты, подбирать изображения и публиковать в Telegram-канал и stories.

## Основные компоненты

### 1. Data Collection
- Telethon
- user account
- выгрузка сообщений из исходных каналов

### 2. AI Layer
- OpenRouter
- модель `google/gemini-2.5-flash`
- генерация постов
- генерация image query только для коротких постов

### 3. Image Layer
- Pixabay API
- поиск изображений
- локальное сохранение картинок
- метаданные в `*.meta.json`

### 4. Publish Layer
- Bot API для постов
- Telethon user account для stories
- `message_id` опубликованного поста сохраняется для story
- основной поток публикует story напрямую из `send_posts.py`

### 5. State / Reliability
- `state.json`
- retry
- alert-уведомления
- lock-файлы
- cleanup

## Бизнес-логика

- Тематика: финансовый канал
- Аудитория: трейдеры, брокеры, инвесторы
- Тон: экспертный, понятный, полезный
- Посты должны быть прикладными и интересными
- Количество постов зависит от насыщенности дня: не растягивать слабые дни на фиксированные 5 постов
- Длинные посты отправляются текстом без изображения
- Короткие посты могут получить релевантную картинку по смысловому image query
- По умолчанию третий пост дня должен быть коротким photo-постом; если AI сделал его длинным, он переписывается через AI до caption limit
- Первый пост дня может триггерить story с кликабельной карточкой опубликованного поста
- На каждом посте может быть CTA-кнопка «Записаться»

## Environments

Есть два окружения:
- `test`
- `prod`

Активное окружение задаётся в:

```json
"env": "test"
```

CLI может переопределить его через `--env`.

## Важно помнить

- истории публикуются через user account
- посты отправляются через bot
- картинки ищутся через Pixabay только для коротких постов
- story использует Telegram media area на опубликованный channel post и видимую карточку, сгенерированную из текста поста
- для story нужен `message_id` из результата публикации; фоновое изображение можно задать в config или использовать автоматический fallback
- алерты шлются отдельным bot token
- cleanup чистит только старые архивы и логи
- generated-папки — это текущая рабочая очередь
